# Contributing Guidelines

Welcome to the Mysteries of the Universe project! These guidelines will help you understand how to structure your contributions, keep the project organized, and ensure a smooth workflow.

## Getting Started

1. **Fork the Repository:**
   - Create a fork of the repository to work on changes.

2. **Clone the Repository:**
   - Clone your forked repository to your local machine.
   ```bash
   git clone https://github.com/yourusername/themyteriesoftheuniverse.git

3. **Creat a Branck:**
   - Create a new branch for your work.
   ```bash
   git checkout -b feature-branch-name

## Making Changes

1. Coding Standards:
  - Follow best practices for HTML, CSS, JavaScript, and other technologies used in the project.
  - Ensure code adheres to W3C standards and uses semantic HTML and ARIA roles/attributes.

2. Commit Messages:
  - Write clear, concise commit messages that describe the changes made.
  - Follow the format: Short description of changes (max 150 characters).

3. Comments:
  - Include comments in your code to explain the purpose of complex sections and any non-obvious implementation details.

## Testing

1. Local Testing:
   - Test your changes locally to ensure they work as expected.
   - Validate your code using appropriate toooks (e.g. HTML validators, CSS validators, etc.).
  
2. Automated Testing
   - If applicable, ensure your changes pass any automated tests set up in the CI/CD pipeline.
  
## Documentation

1. Update Documentation:
   - Update the `README.md` file and any other relevant documentation to reflect your changes.
   - Document any new features, changes to existing features, and any other important information.
  
## Submitting Changes

1. Push Changes:
   -Push your changes to your forked repository.
   ```bash
   git push origin feature-branch-name

2. Pull Request:
   - Create a pull request from your feature branch to the main branch of the original repository.
   - Provide a detailed description of the changes in the pull request, including the purpose and any relevant details.
  
3. Review and Merge:
   - Review the changes to ensure they meet the project's standards.
   - Merge the pull request after ensuring all checks pass and the changes are satisfactory.

## Code of Conduct
By contributing to this project, you agree to abide by the Code of Conduct.
